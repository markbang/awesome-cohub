# Contributing to Awesome Cohub

Thanks for helping curate the Cohub ecosystem.

This repository is a curated list, not a monorepo dump. Prefer high-signal links that help someone start building with Cohub faster.

## What belongs here

- Official Cohub product entry points
- Official CLI / SDK / skills
- Public docs and architecture references
- Community tools, templates, demos, and integrations that clearly relate to Cohub
- High-quality adjacent agent resources that Cohub users actually benefit from

## What does not belong here

- Private or broken links
- Unrelated AI link farms
- Brand-new empty repositories with no docs
- Duplicate entries
- Long marketing paragraphs

## Entry format

Add one line to the relevant section in `README.md`:

```markdown
- **[project-name](https://example.com)** - Short description in about 10 words
```

Rules:

- Keep descriptions short and concrete
- Prefer official sources over mirrors
- Group under the most specific existing section
- If no section fits, propose a new one in the PR

## PR title examples

- `Add resource: cohub-desktop`
- `Docs: clarify Works section`
- `Add skill: public-share`

## Banner / assets

If you improve the header animation:

- keep brand orange `#FF4500`
- keep dark canvas close to `#0B0C10` / `#1F2026`
- update `assets/banner.svg` as the source of truth
- regenerate `assets/banner.gif` and `assets/banner.png` if the motion changes

## Quality bar

Before opening a PR:

1. Click every added link
2. Confirm the project is public and maintained enough to be useful
3. Make sure the description is accurate
4. Keep the README structure clean

## Questions

Open an issue with a clear title. If the question is about Cohub product behavior rather than this list, prefer the main Cohub docs or monorepo.
